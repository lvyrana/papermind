# Memory System Implementation v1

## 1. 当前背景

这一轮改的是 PaperMind 的“系统观察摘要/记忆机制”。

改动原因：

- 旧版只有一条 `interests_summary`
- 它把长期偏好、近期行为、提问倾向混在一段文本里
- 每次重生成都可能丢掉长期特征
- 用户很难理解：这到底是“稳定画像”还是“最近变化”

这次我们把它拆成了更接近“记忆系统”的结构，但目前还处于 **第一版实现 + 尚未充分测试** 的阶段。

所以这份文档的目标不是宣布“已经定型”，而是：

- 记录当前到底改成了什么
- 明确哪些原则已经定下来
- 明确哪些地方还保留调整空间
- 给后续测试和迭代提供稳定基线

## 2. 已经定下来的产品原则

### 2.1 观察记忆不参与检索

这是最重要的边界。

当前原则：

- 检索层只使用用户显式输入
  - `focus_areas`
  - `method_interests`
  - `background`
  - `exclude_areas`
- 观察记忆 **不参与检索生成**

观察记忆只用于：

- 用户对话
- 论文中文解读

当前不再用于：

- 搜索 query 生成
- 分类打分

### 2.2 记忆分成长期和近期两层

当前实现采用：

- `memory_core`
- `memory_recent`

含义：

- `memory_core`：长期稳定骨架
- `memory_recent`：近期关注变化

### 2.3 用户可以手动把近期变化更新到长期画像

当前已加按钮：

- `更新到长期画像`

这不是“编辑一段摘要”，而是：

- 让系统把 `memory_recent` 吸收到 `memory_core`

### 2.4 recent 不参与检索，也不因为改画像就直接清空

这是本轮讨论后明确调整的方向。

原因：

- 测试期用户会频繁改画像
- 如果每次改画像就清空 `recent`，系统会显得反复失忆

所以当前实现方向是：

- `recent` 更像“最近行为的增量观察”
- 不因为画像变化直接失忆

## 3. 旧版机制是什么

旧版主要依赖单字段：

- `interests_summary`

配套字段：

- `interests_summary_updated_at`
- `interests_summary_is_manual`

### 3.1 旧版摘要的用途

旧版摘要：

- 不参与检索
- 参与 AI 对话
- 参与论文解读
- 在“用户手动编辑过摘要”的情况下，也会参与分类

### 3.2 旧版摘要的生成逻辑

旧版 `/api/profile/interests-summary` 会综合：

- 用户填写的画像
- 近期收藏标题
- 收藏分类分布
- 最近跨论文提问

生成一段 150-200 字左右的摘要。

它本质上是：

**一段混合摘要**

把长期和近期信号揉在一起。

### 3.3 旧版的手动编辑/删除逻辑

旧版有一个很重要的行为：

- 如果 `interests_summary_is_manual == "1"`
- 系统就不再自动覆盖摘要

也就是说：

- 用户手动编辑后，系统尊重用户
- 用户手动删除后，也会进入“手动状态”

但旧版还有一个关键补充：

- 一旦画像发生变化
- 后端会清空 `interests_summary`
- 并把 `interests_summary_is_manual` 重置

所以旧版实际效果是：

- **手动保护存在**
- 但不是永久的
- 改画像后，系统又会重新接管摘要生成

## 4. 当前版本已经改成什么样

### 4.1 数据层

当前新增字段：

- `memory_core`
- `memory_recent`
- `behavior_events_since_recent`
- `last_recent_updated_at`
- `last_core_merged_at`
- `core_source`

旧字段仍然保留：

- `interests_summary`
- `interests_summary_updated_at`
- `interests_summary_is_manual`
- `behavior_events_since_summary`

保留原因：

- 兼容旧数据
- 平滑迁移
- 避免一次性删字段引起其它链路出错

### 4.2 memory_core

当前设计：

- 首次保存画像后，如果还没有 `memory_core`
- 系统会自动生成一版长期研究画像

目标：

- 给对话和论文解读一个稳定背景

### 4.3 memory_recent

当前设计：

- 根据最近 **7 天**行为信号生成（recent 窗口 = 一周）
- 主要看：
  - 最近收藏标题
  - 最近跨论文提问
  - 最近阅读轨迹

触发条件：

- 首次没有 recent
- 或累计关键行为 ≥ 8 次
- 或距上次生成 ≥ 7 天且期间有新行为

### 4.4 memory_core 更新周期

- `auto_initial`（首次保存画像生成）：**只要第一次 recent 生成完，立刻刷新**，不等 14 天
- `auto_refresh`（后续）：每 **14 天**自动把最新 recent 整合进 core
- 用户手动：随时点"吸收到长期画像"触发，merge 后 recent 清空

### 4.5 merge-to-core

当前设计：

- 用户点击 `吸收到长期画像`
- 后端读取：
  - 当前 `memory_core`
  - 当前 `memory_recent`
  - 当前明确画像
- 生成一版新的 `memory_core`
- merge 成功后 **清空 recent**，重新开始下一轮 7 天观察

### 4.5 理解层使用方式

当前已经改成：

- AI 对话读取 `memory_core + memory_recent`
- 论文中文解读读取 `memory_core + memory_recent`

当前已经移除：

- 分类对旧摘要的依赖

## 5. 当前代码状态的真实结论

这轮不是“纯讨论”，已经落了代码。

但也不能说“记忆系统已经完全定型”。

更准确地说：

- 结构已经从单摘要升级到双层记忆
- 前后端基本打通了
- 主要边界也已经理顺
- 但还需要真实测试验证用户体验和触发节奏

## 6. 当前最需要测试的点

### 6.1 首次生成 core 的体验

要验证：

- 新用户保存画像后，是否真的会生成 `memory_core`
- 文案是否像“长期研究画像”，而不是另一种摘要

### 6.2 recent 的更新节奏

要验证：

- 是否太勤
- 是否太慢
- 是否会在测试期被噪音行为带偏

### 6.3 merge-to-core 的用户感受

要验证：

- 用户看到 `recent` 后，是否真的理解“更新到长期画像”是什么意思
- merge 后的新 `core` 是否稳定、自然

### 6.4 对话和解读的感知变化

要验证：

- 加入 `core + recent` 后，对话是否更像“理解这个用户”
- 论文解读里的“为什么和你相关”是否更贴近真实关注点

## 7. 当前仍保留调整空间的地方

### 7.1 recent 的观察窗口

当前代码是 14 天。

这是一个比较合理的第一版，但还没经过充分验证。

后续可能调整成：

- 7 天
- 14 天
- 21 天

### 7.2 recent 的更新阈值

当前有事件阈值和时间阈值。

但是否合适，还要看测试：

- 会不会太频繁
- 会不会太迟钝

### 7.3 core 的自动修正力度

当前实现保留了自动低频修正。

这符合“自动 + 手动并行”的方向。

但后续仍需要测试：

- 自动修正会不会过于积极
- 会不会把短期噪音吸收到长期骨架

### 7.4 是否最终删除旧 interests_summary

现在还不适合立刻删。

建议节奏：

1. 先完成几轮真实测试
2. 确认所有前后端链路都已切到 memory 体系
3. 再考虑删除旧字段和旧兼容接口

## 8. 下一步工程实施建议

### 第一步：先不要继续大改逻辑

先做真实测试。

重点测试这几类用户路径：

1. 新用户首次填写画像
2. 已有用户连续阅读/收藏/提问
3. 用户多次调整画像
4. 用户点击“更新到长期画像”

### 第二步：测试后记录现象，不急着重构

建议直接记录：

- 触发时间
- 当时画像内容
- 当时 recent 内容
- merge 前后的 core 内容
- 对话/解读是否变好

### 第三步：基于测试结果再决定下一轮

下一轮可能是：

- 调整 `recent` 触发节奏
- 调整 `core` prompt
- 限制 `recent` 长度
- 决定是否把 old summary 兼容逻辑删掉

## 9. 分支与版本管理建议

这类改动以后建议统一这样做：

1. 新建功能分支
2. 先改数据结构
3. 再改后端接口
4. 再改前端展示
5. 每一层单独 commit
6. 测试完成再合并回主线

当前分支建议：

- `codex/memory-v1`

这样后面即使要回退，也能回得非常清楚。

## 10. 一句话总结

当前这轮的本质不是“优化旧摘要”，而是：

**把单段系统观察摘要，升级成“长期骨架 + 近期变化 + 用户可确认吸收”的第一版记忆系统。**
