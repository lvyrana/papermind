# 精读台右栏重塑 · 设计 Handoff

> 给 Claude Design 的约束卡。目标是把精读页（PaperRead）右栏从"什么都塞"收成一条清晰的主线。
> 你照这份画右栏稿，我照稿一比一接回真实逻辑（同 W2 首页的打法）。
> 作者：雀雀主导 IA 决策；本文档整理现状与后端绑定，供设计对齐。

---

## 0. 北极星

产品当前只做一条简单闭环：**上传 → 精读 → 导出**。
推荐/搜索、记忆地形图等以后再加回来。所以精读台右栏只为这条主线服务：

> **划词读 →（沉淀成卡片）→ 汇报板 → 导出组会 PPT**

右栏是"沉淀 + 出口"，不是"讨论区 + 引导区 + 收藏夹"。

---

## 1. 现状盘点（右栏 = `MemoryChannel` 巨型组件）

当前一根 420px 的右轨里塞了 6 块，这就是要精简的对象：

| 模块 | 作用 | 主线角色 |
|---|---|---|
| 对话 / 追问 (chat) | 和 AI 讨论论文 | 手段，非产物 |
| 阅读卡片 (cards) | 四类结构化沉淀 | **产物核心** |
| 带读 / 陪读引导 | 路线图 / 摘要带读 / 当前页陪读 | 引导，可收敛 |
| 自由笔记 (notes) | 边读边记 | 沉淀的一种 |
| 汇报板 + 导出 (board) | 组会 PPT 出口 | **出口核心** |
| 收藏 / 项目 (bookmark) | 收藏、归项目 | 元操作，可挪走 |

---

## 2. 极简 IA 提案（大方向已认，粗体处是"你要调"的自由区）

**常驻右栏（主角）：**
- **① 沉淀区** —— 阅读卡片列表（可含笔记）。你要导出的东西住这里。
- **② 汇报板 + 导出** —— 板块×条目数的紧凑区块 + "打开全览" + "导出 PPT"，钉在底部。

**降级为按需唤出（不常驻占轨）：**
- **对话 / 追问**：划词浮窗，或一个按钮弹抽屉 —— *你要调：是否保留一小块常驻？*
- **带读 / 陪读引导**：收成顶部一个"怎么读这篇"入口 —— *你要调：精简成一个入口，还是保留三态？*
- **自由笔记**：并进沉淀区，还是独立一块 —— *你要调*
- **收藏 / 项目**：挪进标题栏 ⋯ 溢出菜单 —— *你要调：留轨里还是挪走？*

> 一句话：**右栏 = 沉淀 + 导出，其余"用时才出现"。** 具体常驻/按需的边界，画的时候由你定。

---

## 3. 必须保留的能力 + 后端绑定（设计里别把入口弄丢）

这些能力后端已存在，无论摆哪，入口都得能接回去：

| 能力 | 说明 | 后端接口 |
|---|---|---|
| 划词 → quote | 选中 PDF 文字灌成引用，**和 PDF 高亮双向联动**（删 quote 会擦掉对应高亮） | 前端派生 + localStorage/后端 chat |
| 阅读卡片 CRUD | 增删改查，每张**带原文页码锚点可跳转** | `POST/GET/PATCH/DELETE /api/cards` |
| 划词/卡片/对话 → 送到汇报 | 投进汇报板某板块（有默认映射，选单可改投） | `POST /api/board/{id}/items`（板块选单 `BoardSectionPicker`） |
| 汇报板导出 | 导出 marp 格式的组会 PPT | `GET /api/board/{id}/export/marp` |
| 对话追问 | 和 AI 讨论，回复可"归卡"、可"送到汇报" | `POST /api/chat` |
| 带读/陪读 | roadmap / summary / current-page 三种引导 | 精读引导接口（`deepReadMode`） |
| 笔记 | 自由笔记，AI 对话可保存为笔记；可导出 Markdown | `/api/notes` |
| 收藏 / 项目 | bookmark、归入 project | `/api/library`、`/api/projects` |

## 4. 真实内容形态（mock 请用真数据结构，别编字段）

**阅读卡片 — 四类**（颜色沿用 token）：

| key | 标签 | 配色 |
|---|---|---|
| method | 方法 | coral |
| finding | 发现 | mint |
| critique | 批判 | navy |
| transfer | 迁移 | coral-deep |

每张卡片：`标签 · 标题 · 正文 · 原文页码锚点（可跳 PDF）`。

**汇报板 — 七板块**（默认，空板块灰显＝还没读到）：

`研究背景 / 研究问题 / 方法 / 关键结果 / 批判与局限 / 对我的启发 / 讨论问题`

**卡片 → 板块 默认映射**：方法→方法、发现→关键结果、批判→批判与局限、迁移→对我的启发。

**导出**：汇报板 → marp → 组会 PPT（`downloadBoardMarp`）。

## 5. tokens / 尺寸 / 交互约束

- **配色**：沿用项目 token —— `cream #F7F0E8` / `cream-dark #EDE4D8` / `navy #1E3A5F` / `navy-light #2D5380` / `coral #E8877A` / `coral-light #F0A89E` / `mint #A8D5BA` / `mint-deep #7BB89C` / `warm-white #FFFDF9` / `warm-gray #8E8A85`。
- **字体**：serif=Noto Serif SC（标题/问候），sans=DM Sans（正文）。
- **右栏尺寸**：当前 420px，带 resize 手柄（可拖窄）；精简后可以更窄，宽度不设死。
- **三栏语境**：左(240 元数据/TOC) ｜ 中(PDF) ｜ 右(本次要重塑的)。移动端 <1024 退化为单栏 tab（pdf/meta/chat），设计以桌面为主，移动端我按现有 tab 逻辑接。
- **精读不被打断**：右栏空态别硬占位（现状"只在有 quote 时才展示追问区"就是这个原则），沉淀区没内容时给安静的引导而非大空框。
