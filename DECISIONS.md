# 产品决策记录

记录 PaperMind 开发过程中的重要产品决策，包括选择的方案、放弃的方案和原因。

---

## 2026-03-31

### 无登录 + 设备隔离方案
**决策**: 不做用户登录，用 UUID 存 localStorage 实现设备隔离。  
**原因**: 测试阶段降低摩擦，用户打开即用。每台设备有独立数据，隐私更好。  
**代价**: 换设备或清除浏览器数据会丢失记录；无法跨设备同步。  
**未来**: 正式版考虑加可选登录（微信/邮箱），登录后可绑定设备 UUID 迁移数据。

### 内置 API Key，用户无需配置
**决策**: 把 API Key 放在后端 .env，前端 Settings 页简化为说明卡片。  
**原因**: 测试阶段朋友打开直接可用，不用被 API 配置劝退。  
**代价**: API 消耗由作者承担；不适合大规模开放。  
**未来**: 正式版开放自定义 API 配置，或转为订阅模式。

### LLM 优先级：阿里云通义 → GLM → DeepSeek
**决策**: 阿里云 DashScope 作为主力（免费额度大，模型质量好），后两者兜底。  
**原因**: 阿里云赠送了大量免费 token，qwen-flash/plus 质量明显优于 GLM-4-flash。  
**备注**: 可在 .env 中切换 QWEN_MODEL，目前测试 qwen-flash，生产考虑 qwen-plus。

### Rate Limit：普通用户 20次/天，owner 不限量
**决策**: 每人每天最多 20 次论文推荐，OWNER_UID 设备不限量。  
**原因**: 防止 API 被滥用；作者本人测试需要不限量。  
**实现**: .env 中设置 OWNER_UID，Settings 页展示设备 ID 方便复制。

## 2026-03-30

### 论文搜索：LLM 生成关键词，替代硬编码
**决策**: 不再硬编码 COPD 关键词，改为根据研究者画像 + 收藏历史，让 LLM 生成 PubMed 检索词。  
**原因**: 硬编码无法适应不同研究方向；画像驱动的检索结果更个性化，随使用积累越来越准。  
**效果**: 切换研究方向后，推荐论文立即跟随变化。

### 动态分类：LLM 打分 + 生成标签，替代硬编码分类
**决策**: 移除固定的六个 COPD 分类，改为 LLM 对每批论文打分（0-10）+ 动态生成分类标签。  
**原因**: 固定分类换了研究方向就失效；动态标签更贴合实际内容。  
**代价**: 每次推荐多一次 LLM 调用；标签命名不完全一致。

### 排除领域：双重过滤
**决策**: 排除领域在两处生效：① 生成关键词时不产生相关检索词；② 打分时强制 0 分。  
**原因**: 单一过滤不够——搜索词层面不拦截会抓来不想看的论文；打分层面不强制会仍然混入低分但非零分的内容。

### 异步抓取 + 前端轮询
**决策**: `/api/papers` 触发后台线程抓取，立即返回 `{loading: true}`，前端每 3 秒轮询。  
**原因**: 原来的同步模式下，用户离开首页会中断 fetch，回来后重新触发，浪费次数且体验差。  
**效果**: 抓取在后台完成，导航到其他页面再回来，结果已就绪。

## 2026-03-29

### 视觉风格：暖色系 + 呼吸感动画
**决策**: 主色调用暖白/米色/coral，避免大面积渐变，动画用 breathe-in 轻微入场。  
**原因**: 学术工具不应压迫感太重；暖色调比冷蓝色更有"陪伴感"。  
**原则**: 渐变只用于强调元素，不用于背景大面积铺色。

### 摘要翻译：按需触发，不自动翻译
**决策**: 摘要默认显示英文原文，点击"译"按钮后才请求翻译，可切换原文/译文。  
**原因**: 研究者通常能读英文摘要；自动翻译会消耗大量 API；按需触发节省资源，也不干扰习惯读原文的用户。

## 2026-03-27

### AI 解读的 relevance 字段
**决策**: relevance 只说"这篇论文的哪个发现/方法能给研究者带来什么具体启发"，不罗列研究者画像关键词。  
**原因**: 早期 GLM-4-flash 会把用户所有研究兴趣都列一遍，显得机械；用户明确反馈"笨笨的"。  
**Prompt 要点**: 明确告知"不要罗列研究者画像关键词"，且"如果论文没有直接涉及某个方向就不要提"。
